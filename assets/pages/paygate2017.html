Payment & Refund Enhancement for a ticket commerce site

Jul 2017, remote

‹ Previous › Next

Introduction & Goal:

This project had two goals. One is replacing an old payment module that supports only IE browsers with a cross-platform module; it led to enhance browser compatibility. The other is made the user experience better by modifying the refund process, which is processed by manually, to do automatically.

Role:

Analyzed the payment and refund process. Integrated with a PayGate(PG) API and a Web plugin. Modified a payment database schema to store payment reference numbers returned from the PG.

Result:

Reduced customer complaints about the unable to pay in the Chrome browsers. Dramatically decreased customer inquiries of the refund process.
  • Front-end: jQuery, Bootstrap4
  • Back-end: PHP5.3, Linux, MySQL, Crontab
